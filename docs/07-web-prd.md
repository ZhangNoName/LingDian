## 2. 文档目标

本文件用于明确 Web 端应该具备哪些核心界面、页面职责、模块边界和优先级，帮助后续完成：

- 后台信息架构设计
- 菜单与路由设计
- 原型设计
- 前后端接口拆分
- 开发排期评估

## 3. Web 端定位

Web 端不只是“展示官网”，更建议承担整个点餐系统的运营和管理职责，作为 B 端核心工作台使用。

建议 Web 端分为两类能力：

- 经营管理后台：给总部、门店店长、店员、运营人员使用
- 门店辅助工作台：给收银、出餐、订单处理、库存管理等角色使用

因此 Web 端页面体系应围绕以下业务目标建设：

- 管理门店与营业配置
- 管理商品、分类、套餐、价格、库存
- 管理订单履约与退款售后
- 管理用户、会员、优惠券、营销活动
- 查看经营数据与统计报表
- 管理仓库、原料、采购、出入库

## 4. 用户角色

### 4.1 平台管理员

职责：

- 管理品牌级配置
- 管理组织架构、角色权限
- 查看全局经营数据

### 4.2 运营人员

职责：

- 管理活动、优惠券、首页推荐位
- 管理门店展示内容、专题活动

### 4.3 店长

职责：

- 管理门店营业状态
- 管理菜品上下架与库存
- 处理异常订单、退款、售罄
- 查看门店经营数据

### 4.4 店员/收银员

职责：

- 查看订单
- 操作接单、出餐、叫号、核销
- 处理部分售后

### 4.5 仓库管理员

职责：

- 管理原料库存
- 处理采购入库、调拨、盘点、损耗

## 7. 核心模块详细需求

## 7.1 工作台

### 页面目标

让不同角色在登录后第一时间看到“今天最重要的信息”和“当前待处理事项”。

### 页面组成

- 顶部经营概览卡片
- 今日销售额、订单数、客单价、退款数
- 待处理事项
- 门店营业状态提示
- 缺货/低库存提醒
- 近 7 日趋势图
- 快捷入口

### 关键交互

- 不同角色看到不同工作台内容
- 点击卡片可进入对应模块详情页
- 可按门店和日期范围筛选

## 7.2 门店管理

### 目标

统一管理门店基础资料和营业规则。

### 核心页面

#### 1. 门店列表页

功能：

- 查看所有门店
- 按状态筛选
- 按城市、区域、门店类型筛选
- 支持新建和启停用

字段建议：

- 门店名称
- 门店编号
- 所属品牌
- 营业状态
- 堂食/外带/自提开关
- 今日订单数
- 操作

#### 2. 门店详情页

功能：

- 查看门店完整资料
- 查看营业信息、门店图片、联系人、配送设置
- 查看门店关联菜单和活动

#### 3. 营业设置页

功能：

- 设置营业时间
- 设置节假日安排
- 设置暂停营业
- 设置接单规则

#### 4. 桌台码管理页

功能：

- 管理桌号
- 生成桌台二维码
- 绑定区域和容纳人数

## 7.3 商品与菜单

### 目标

完成商品全生命周期管理，包括分类、规格、套餐、上下架和库存联动。

### 核心页面

#### 1. 商品列表页

功能：

- 查看商品列表
- 按分类、上架状态、门店、标签筛选
- 批量上下架
- 批量改价
- 批量设置推荐

建议字段：

- 商品图
- 商品名称
- 分类
- 售价
- 成本价
- 库存状态
- 销量
- 上架状态
- 操作

#### 2. 商品编辑页

功能：

- 编辑商品名称、描述、图片
- 设置价格、规格、加料
- 设置标签
- 设置是否参与活动
- 设置堂食/外带可售范围

#### 3. 商品详情页

功能：

- 查看商品完整信息
- 查看销售趋势
- 查看关联订单与用户评价

#### 4. 分类管理页

功能：

- 分类新增/编辑/排序
- 设置分类显示范围

#### 5. 菜单管理页

功能：

- 创建菜单版本
- 绑定门店
- 配置生效时间
- 支持午市/晚市菜单

#### 6. 套餐管理页

功能：

- 创建套餐
- 绑定主商品与可选项
- 配置套餐价格和优惠展示

#### 7. 库存状态页

功能：

- 查看售罄商品
- 手动售罄/恢复
- 联动库存提醒

## 7.4 点餐与订单

### 目标

让门店或运营人员可以实时处理订单、跟踪履约、查看订单详情和处理售后。

### 核心页面

#### 1. 订单列表页

功能：

- 查看订单列表
- 按订单类型筛选：堂食 / 外带 / 自提
- 按订单状态筛选
- 按支付状态筛选
- 按时间、门店、渠道筛选

建议字段：

- 订单号
- 门店
- 用户
- 订单类型
- 商品摘要
- 应付金额
- 支付状态
- 履约状态
- 创建时间
- 操作

#### 2. 订单详情页

功能：

- 查看订单完整信息
- 查看商品明细、优惠明细、支付信息
- 查看用户信息和履约记录
- 支持补打小票、发起退款、备注异常

重点模块：

- 基础信息区
- 商品明细区
- 金额明细区
- 操作日志区
- 售后记录区

#### 3. 出餐/叫号工作台

功能：

- 实时刷新待处理订单
- 支持接单、制作中、待取餐、已完成
- 支持叫号和核销

#### 4. 售后处理页

功能：

- 查看退款申请
- 审核通过/驳回
- 记录退款原因
- 追踪退款状态

#### 5. POS/收银辅助页

功能：

- 门店前台快速录单
- 现金/扫码收款登记
- 堂食桌台关联

## 7.5 用户与会员

### 目标

帮助运营和门店识别高价值用户，承接会员体系和用户资产。

### 核心页面

#### 1. 用户列表页

功能：

- 查看用户基础资料
- 按注册时间、活跃度、消费金额筛选
- 查看标签和会员状态

#### 2. 用户详情页

功能：

- 查看基础信息
- 查看订单记录
- 查看优惠券、积分、余额
- 查看行为标签

#### 3. 会员等级页

功能：

- 定义等级规则
- 设置权益内容
- 关联折扣与成长值

#### 4. 积分与优惠券页

功能：

- 查看发放记录
- 手动补发
- 调整积分

## 7.6 营销中心

### 目标

支持活动配置和转化提升。

### 核心页面

#### 1. 活动列表页

功能：

- 查看活动状态
- 按门店、时间、类型筛选
- 上下线活动

#### 2. 活动编辑页

活动类型建议：

- 满减
- 折扣
- 新人券
- 套餐优惠
- 加价购
- 限时秒杀

编辑内容：

- 活动名称
- 活动时间
- 适用门店
- 适用商品
- 优惠规则
- 领取规则
- 展示规则

#### 3. Banner/推荐位配置页

功能：

- 配置小程序首页 Banner
- 配置推荐商品位
- 设置跳转链接和排序

## 7.7 统计分析

### 目标

让管理者通过 Web 端看到经营效果，而不是只做操作后台。

### 核心页面

#### 1. 经营总览页

指标：

- 销售额
- 订单数
- 客单价
- 支付转化率
- 退款率

#### 2. 订单统计页

指标：

- 各状态订单占比
- 渠道来源占比
- 堂食/外带/自提占比

#### 3. 商品分析页

指标：

- 销量排行
- 销售额排行
- 毛利排行
- 滞销商品

#### 4. 用户分析页

指标：

- 新增用户
- 活跃用户
- 复购率
- 会员消费占比

#### 5. 活动分析页

指标：

- 活动曝光
- 领券量
- 核销量
- 活动订单金额

#### 8. 采购单页

### 10.1 商品管理链路

商品列表页 → 商品编辑页 → 商品详情页 → 关联菜单页 → 上下架/库存状态页

## 12. 非功能要求

### 12.1 权限要求

- 必须支持角色权限控制
- 必须支持门店维度的数据隔离
- 必须支持按钮级权限

### 12.2 性能要求

- 列表页支持分页与筛选
- 大屏统计页支持缓存
- 订单工作台支持近实时刷新

### 12.3 易用性要求

- 常用操作不超过 3 次点击
- 高优先级状态使用明显颜色和提醒
- 列表页必须支持导出

## 13. 当前结论

对这个项目来说，Web 端建议定义为“餐饮经营管理后台”，而不是单纯官网。  
从页面建设上，至少应包含以下 8 大类界面：

- 工作台
- 门店管理
- 商品与菜单
- 点餐与订单
- 用户与会员
- 营销中心
- 统计分析
- 仓库管理

如果你现在准备正式做 Web 后台，最先应该落地的是：

1. 工作台
2. 门店管理
3. 商品与菜单
4. 订单中心
5. 基础统计
6. 系统设置

这些页面就足够支撑第一版管理端上线。
